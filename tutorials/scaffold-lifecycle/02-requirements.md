# Chapter 2: Requirements

## What Happens

The `/requirements` skill helps you define what the trading app needs to do.

## Core Features to Request

When asked what the app should do, describe these features:

1. **Marketplace browsing** — List and search marketplace offerings
2. **DEX quotes** — Get swap quotes from multiple decentralized exchanges
3. **Wallet management** — Check balances, view transaction history
4. **Trade execution** — Execute swaps through the DEX aggregator
5. **API key authentication** — Protect endpoints with API keys

## Expected User Flow Diagrams

The requirements phase should generate at least 3 flow diagrams:

1. **Marketplace browsing flow** — User searches → views listings → (optionally) purchases
2. **DEX swap flow** — User requests quote → reviews options → executes swap → confirmation
3. **Wallet management flow** — User checks balance → views history → (optionally) sends XRP

## Expected Output

- `docs/requirements.md` with user stories and mermaid flow diagrams
- Your approval before moving on
